1. 图的基本概念

图(Graph)——图G是两个集合V（G）和E（G）组成的，记为G = (V,E)其中V（G）是顶点的非空有限集，E（G）是边的有限集合，边是顶点的无序对或有序对。

  • 有向图
    E是有向边（也称为弧）的有限集合，记为<v,w>
  • 无向图
    E是无向边的有限集合，边是顶点的无序对，记为(v,w)
  • 有向完全图
    n个顶点的有向图最大边数是n(n-1)
  • 无向完全图
    n个顶点的无向图最大边数是n(n-1)/2
  • 邻接点
    若(vi,vj)是一条无向边，则称vi vj互为邻接点
  • 关联
    vi和vj互为邻接点，称边(vi,vj)关联于顶点vi和vj
  • 顶点的度
    无向图中，顶点的度为每个顶点相连的边数；
    在有向图中，分为入度和出度，度 = 出度+入度。
    在有向图中，箭头是具有方向的，从一个顶点指向另一个顶点，这样一来，每个顶点被指向的箭头个数，就是它的入度。从这个顶点指出去的箭头个数，就是它的出度。
  • 路径
    路径是顶点的序列
  • 路径长度
    沿路径边的数目或沿路径各边权值之和
  • 回路
    第一个顶点和最后一个顶点相同的路径叫做回路
  • 简单路径
    序列中顶点不重复出现的路径叫做简单路径
  • 简单回路
    除第一个和最后一个顶点外，其余顶点不重复出现的回路叫做简单回路
  • 图的连通性
    在图论中，连通图基于连通的概念。在一个无向图 G 中，若从顶点 i 到顶点 j 有路径相连（当然从j到i也一定有路径），则称 i 和 j 是连通的。如果 G 是有向图，那么连接i和j的路径中所有的边都必须同向。如果图中任意两点都是连通的，那么图被称作连通图。如果此图是有向图，则称为强连通图（注意：需要双向都有路径）。图的连通性是图的基本性质。
  • 权
    与图的边或弧相关的数叫权
  • 网
    带权的图叫网
  • 生成树
    一个连通图的生成树是一个极小连通子图，它含有图中全部顶点，但只有足以构成一棵树的n-1条边；一颗有n个顶点的生成树有且仅有n-1条边
  • 自由树（无根树）
    无回路的的图称为树或自由树或无根树
  • 有向树
    只有一个顶点的入度为0，其余顶点的入度为1的有向图
  • 有关图的基本操作

  • 对于无向图
    邻接矩阵一定是个对称矩阵，可压缩存储；有n个顶点的无向图需存储空间n(n+1)/2,行（列）非零元素个数表示度；
  • 对于有向图
    矩阵不一定是一个对称矩阵，有n个顶点的有向图需存储空间为n^2,行非零元素的个数表示出度，列非零元素个数表示入度。
    邻接矩阵的存储空间只和顶点个数有关，和边数无关。

2.3 邻接表

对于图中每个顶点建立一个邻接关系的单链表，并将其表头指针用向量（一维数组）存储，该结构成为邻接表。
无向图的第i个链表将图中与顶点vi相邻接的所有顶点链接起来，也就是链表中的表头节点到链表中的每个节点表示了与表头节点vi相关的边。
有向图的第i个链表，链接了以顶点vi为弧尾（射出）的所有顶点，也就是链表中表头节点到链表中每个节点表示了图中关联到表头结点vi的所有弧。


若问题总对入度更关心，则可以把线性链表的表节点的数据域更改即可；由表尾改为表头，所有以头结点为表头的弧组成的线性链表。

3. 图的遍历

图的遍历是从图的某个顶点出发，按照某种方式系统的访问图中所有的顶点，使每个顶点仅被访问一次。图的遍历通常有两种方式：深度优先搜索和广度优先搜索，他们对有向图和无向图都适用。

3.1 深度优先遍历（DFS）

  • 过程
    1. 从任一个顶点v出发，访问该顶点； 2. 然后依次从v的未被访问的邻接点出发深度优先遍历图，直至途中所有和v有路径相通的顶点都被访问到； 3. 此时若图中有顶点未被访问，则另选中下一个未被访问的顶点做起始点，访问该顶点，继续过程2，直到所有顶点都被访问完。
    ![alt](./../resource/Graph/8.png) **注**：v8访问完之后由于是有向图所以最后访问v5，如果是无向图则会v8 -> v5这个顺序 ![alt](./../resource/Graph/9.png)
  • 算法

3.2 广度优先遍历（BFS）

1. 从图中某顶点出发，在访问v之后 2. 依次访问v的各个未曾被访问的邻接点，然后分别从这些邻接点出发依次访问他们未被访问的邻接点，并使“先被访问的顶点的邻接点”先于“后被访问的顶点的邻接点”被访问，直至图中所有已经被访问的顶点的邻接点都被访问到； 3. 若图中尚有顶点未曾被访问，则另选图中一个未曾被访问的顶点做起点访问该节点，继续过程2，直至所有顶点都被访问完毕。

4. 图的连通性问题

  • 连通图的生成树

    • 设G=（V，E）是一个连通图，则从图中任一顶点出发遍历图时，必将E（G）分成两个集合T（G）和B（G）。其中T（G）是遍历图时所经过的边集，B（G）是剩余的边集。考虑图G’=（V，T），由于V（G’）=V（G），E（G’）=E（G）， 则G’是G的连通子图，它包括G的所有顶点和足以构成一棵树的n-1条边，称为G的生成树（生成树中不存在回路）。
    • 分为深度优先生成树和广度优先生成树
    • 生成森林：非连通图每个连通分量的生成树一起组成的非连通图生成树
    • 说明

      一个图可以有许多棵不同的生成树
      所有生成树具有以下共同的特点：

      • 生成树的顶点个数与图的顶点个数相同
      • 生成树是图的极小连通子图
      • 一个有n个顶点的连通图的生成树一定有n-1条边
      • 生成树中任意两个顶点间的路径是唯一的
      • 在生成树中再加一条边必然形成回路

  • 最小生成树（MST）

  • 问题引入：

  • 性质：
    设N=（V，E）是一个网络，U是顶点集合V的一个真子集，集合W=V-U，如果（u，v）的顶点u$\in$U,v$\in$W,且边（u，v）是图N中所有一个端点在U里，一个端点在W里的边中权值最小的边，则一定存在N的一颗最小生成树包括此边（u,v）。

  • 构造方法
    1.普里姆Prim算法（图使用邻接矩阵实现）

    • 设N = （V，E）是联通网，TE是N上最小生成树中边的集合
    • 初始令顶点集合U = ${u_0},(u_0\in V),TE = \Phi$
    • 在所有u$\in$U,v$\in$V-U的边（u，v）中，找一条代价最小的边（$u_0,v_0$）
    • 将（$u_0,v_0$）并入集合TE，同时v_0并入U
    • 重复上面的操作直至U=V为止，，则T=（V，TE）为N的最小生成树

    2.克鲁斯卡尔（Kruskal）

    • 初始状态为只有n个顶点而无边的非连通图T=（V,$\Phi$），每个顶点自成一个连通分量
    • 在E中选取代价最小的边，若该边依附的顶点落在T中不同的连通分量上，则将此边加入到T中；否则，舍去此边，选取下一条代价最小的边
    • 以此类推，直至T中所有顶点都在同一连通分量上

5. 有向无环图及其应用

问题引入

  • AOV网
    用顶点表示活动，用弧表示活动间优先关系的有向图成为顶点表示活动的网（Activity On Vertex network），简称AOV网
    • 若<$v_i,v_j$>是图中有向边，则$v_i$是$v_j$的直接前驱，$v_j$是$v_i$的直接后继
    • AOV网中不允许有回路，否则意味着某项活动以自己为先决条件
  • 拓扑排序
    把AOV网络中各顶点按照他们相互之间的优先关系排列成一个线性序列的过程叫做拓扑排序。
    • 作用：检测AOV网中是否存在环，对有向图构造其顶点的拓扑有序序列，若网中所有顶点都在他的拓扑有序序列中，则该AOV网必定不存在环
    • 方法
      • 在有向图中选一个没有前驱的（入度为0）的顶点且输出之；
      • 从图中删除该顶点和所有以它为尾的弧
      • 重复上述两步，直至所有顶点均已输出；或者当图中不存在无前驱的顶点为止
    • 算法实现
      • 以邻接表作存储结构
      • 把邻接表中所有入度为0的顶点进栈
      • 栈非空时，输出栈顶元素$v_j$并退栈；在邻接表中查找$v_j$的直接后继$v_k$,把$v_k$的入度减1；若$v_k$的入度为0则进栈；
      • 重复上述操作直至栈为空为止。若栈空时输出的顶点个数不是n，则有向图有环；否则，拓扑排序完毕。
  • 关键路径
    • AOE网(Activity On Edge)：也叫边表示活动的网。是一个带权的有向无环图，其中顶点表示事件，弧表示活动，权表示活动持续时间；
    • 路径长度：路径上个活动持续时间之和
    • 关键路径：从开始顶点到完成顶点的路径中长度最长的叫关键路径
    • 关键活动：关键路径上的活动叫关键活动
    • 寻找关键活动
      设活动$a_k$用弧<i,j>表示，其持续时间记为：dut(<i,j>)则有：
      （1）e（k）= ee(i)
      （2）l(k) = le(j)-dut(<i,j>)
      求ee（i）和le（j）
      （1）从ee（1）= 0开始向前递推
      $ee(j) = Max{ee(i) + dut(<i,j>)},<i,j>\in T,2 \le j \le n$
      其中T时所有以j为头的弧的集合
      （2）从le(n)=ee（n）开始向后递推
      $le(i) = Min{le(j) - dut(<i,j>)},<i,j>\in S,1 \le j \le n-1$
      其中S是所有以i为尾的弧的集合
    • 算法描述
      • 输入顶点和弧信息，建立其邻接表
      • 计算每个顶点的入度
      • 对其进行拓扑排序
        • 排序过程中求顶点的ee[i]
        • 将得到的拓扑序列进栈
      • 按逆拓扑序列求顶点的le[i]
      • 计算每条弧的e[k]和l[k]，找出e[k] = l[k]的关键活动
